# Valuation Notes

## Build-equivalent value

A serious rebuild of this system would require prediction logic, Streamlit pages, odds integration, CSV processing, audit logic, local profile support, proof-ledger mechanics, dashboards, security checks, and documentation.

A reasonable build-equivalent range for a capable prototype is roughly $30k-$100k depending on polish, testing, and API coverage.

## What supports a $100k+ valuation

A $100k+ valuation becomes more defensible when the product has:

1. Timestamped proof ledger.
2. Valid hash-chain verification.
3. Positive ROI over a large sample.
4. Public results dashboard.
5. Multi-user readiness.
6. Clear security and compliance documentation.
7. Paying users or strong user demand.

## What does not support a high valuation alone

- Screenshots without a ledger.
- Win rate without odds.
- Small sample size.
- Manually edited results without audit rules.
- A private prototype with no users or revenue.

## Key buyer metric

Positive ROI matters more than raw win rate. A system hitting 70%-80% but only on heavy favorites may still be unprofitable. The proof ledger must show both win rate and units/ROI.
